## HTML とは

HTML（**H**yper**T**ext **M**arkup **L**anguage、ハイパーテキスト・マークアップ・ランゲージ）は、ウェブサイトのコンテンツ構造を作成するためのコードです。コンテンツは段落、箇条書きリスト、画像、データテーブルなどの要素で構成されます。

HTML は「要素」という部品で構成されています。これらの要素を使って、コンテンツのさまざまな部分を囲み、特定の見た目や機能を持たせることができます。

例えば、「タグ」という特別な記号を使うと、次のようなことができます。

- 単語や画像にリンクを付ける
- 文字を斜めに表示する
- 文字の大きさを変える

このように、HTML を使えば様々な方法でウェブページのコンテンツを整理し、表現することができます。

## HTML 要素

### 1. **基本構造**

HTML 要素は通常、開始タグと終了タグで囲まれた形式をとります。例えば、段落を表す`<p>`要素の構造は以下のようになります。

```html
<p>ここに段落のテキストが入ります。</p>
```

- **開始タグ**: `<p>`
- **コンテンツ**: `ここに段落のテキストが入ります。`
- **終了タグ**: `</p>`

一部の HTML 要素（例: `<img>` や `<br>`）はコンテンツを持たず、「空要素（self-closing elements）」と呼ばれます。

---

### 2. **入れ子構造（ネスト）**

HTML では、要素を他の要素の中に入れることができます。これを「入れ子構造」と呼びます。ページのセクションを整理し、レイアウトを作成するために役立ちます。

例えば、段落の中にテキストを強調したい場合は、以下のように`<strong>`要素を使って入れ子にできます。

```html
<p>HTMLは <strong>とても重要な</strong> 技術です。</p>
```

### 3. **セマンティック構造**

HTML5 から追加されたセマンティック要素（例: `<header>`、`<footer>`、`<section>`）により、ページの構造を意味に基づいて整理できます。これにより、スクリーンリーダーや検索エンジンがページの構造を理解しやすくなります。

```html
<article>
  <header>
    <h1>記事のタイトル</h1>
    <p>公開日: 2024年11月</p>
  </header>
  <section>
    <p>これは記事の本文です。HTMLについての解説が含まれています。</p>
  </section>
  <footer>
    <p>著者: John Doe</p>
  </footer>
</article>
```

この例では、`<article>` 要素の中に、`<header>`、`<section>`、`<footer>` が入っており、記事全体の構造をわかりやすくしています。

---

### 4. **属性（Attributes）**

HTML 要素には、要素の特性を指定する「属性」を含めることができます。属性は、開始タグの中で設定され、追加の情報を提供します。

```html
<a href="https://example.com" target="_blank">Exampleサイトへ</a>
```

- **href**: `<a>` タグでリンク先を指定します。
- **target**: リンクを新しいタブで開くかどうかを指定します。

---

### 5. **コメント**

HTML 内でメモや説明を書きたい場合は、コメントを使います。コメントはブラウザには表示されません。

```html
<!-- これはコメントです -->
<p>HTMLについて学びましょう。</p>
```

---

このように、HTML の構造は「開始タグと終了タグでコンテンツを囲む」「他の要素の中に入れ子にする」「属性で追加情報を指定する」ことで成り立っています。これらを使いこなすと、読みやすく構造化されたページを作ることができます。

## HTML タグの構造

HTML は「タグ」で構成されており、タグは角括弧 `< >` で囲まれたキーワード（タグ名）で始まります。多くのタグは、開くタグと閉じるタグのペアで構成され、閉じるタグにはスラッシュ（`/`）が付いています。例えば、段落を作成する `<p>` タグは以下のように書きます。

```html
<p>これは段落です。</p>
```

一部のタグは「セルフクローズドタグ」と呼ばれ、閉じるタグが不要です。たとえば、画像を表示する `<img>` タグなどが該当します。

```html
<img src="image.jpg" alt="画像の説明" />
```

## HTML の基本構造

HTML ドキュメントの基本構造には、次の主要部分があります。

- **`<!DOCTYPE html>`**: HTML5 のドキュメントであることを示します。
- **`<html>`**: HTML ドキュメントのルート要素です。`lang` 属性を指定してドキュメントの言語を設定します。
- **`<head>`**: メタ情報やスタイルシート、JavaScript ファイルなど、ページのヘッダー情報を含みます。
  - `<title>`: ページのタイトルを設定します。検索結果やブラウザタブに表示されます。
  - `<meta charset="UTF-8">`: 文字エンコーディングを設定し、一般的に `UTF-8` が推奨されます。
- **`<body>`**: ページのメインコンテンツを含む部分で、ユーザーに表示されるすべての要素がここに書かれます。

## 主な HTML 要素

- **見出し要素** (`<h1>`, `<h2>`, `<h3>`, `<h4>`, `<h5>`, `<h6>`, .): 段階的な見出しを作成します。`<h1>`が最も重要で、通常ページのタイトルとして使用されます。

  ```html
  <h1>これはメインタイトル</h1>
  <h2>サブタイトル</h2>
  ```

- **段落要素** (`<p>`): テキストを段落として表示します。

  ```html
  <p>これは段落の例です。</p>
  ```

- **リンクタグ** (`<a>`): 別のページや場所へのリンクを作成します。`href` 属性でリンク先を指定します。

  ```html
  <a href="https://example.com">Exampleサイトへ</a
  ```

- **画像タグ** (`<img>`): 画像を表示します。`src` 属性で画像の URL を、`alt` 属性で代替テキストを指定します。

  ```html
  <img src="image.jpg" alt="画像の説明" />
  ```

- **リストタグ**:

  - **順序付きリスト** (`<ol>`): 番号付きのリストを作成します。

    ```html
    <ol>
      <li>アイテム1</li>
      <li>アイテム2</li>
    </ol>
    ```

  - **順序なしリスト** (`<ul>`): 順序のない箇条書きリストを作成します。

    ```html
    <ul>
      <li>アイテムA</li>
      <li>アイテムB</li>
    </ul>
    ```

- **テーブルタグ** (`<table>`): 表形式のデータを表示します。

  - `<table>`：表全体を表すタグです。
  - `<tr>`：表の行を作成します。
  - `<td>`：表のセルを作成します。
  - `<th>`：見出しセルを作成し、通常は太字になります。

    ```html
    <table>
      <tr>
        <th>ヘッダー1</th>
        <th>ヘッダー2</th>
      </tr>
      <tr>
        <td>データ1</td>
        <td>データ2</td>
      </tr>
    </table>
    ```

- **フォームタグ**(`<form>`):ユーザーからの情報を入力してもらう場所で、サーバーに送信するために使用されます。

  - `<form>`：フォーム全体を囲む要素で、データの送信先（`action` 属性）や送信方法（`method` 属性）を指定します。
  - `<input>`：ユーザーがデータを入力できるフォームフィールドです。
  - `<button>`：フォーム送信用ボタンです。

    ```html
    <form action="/submit" method="POST">
      <input type="text" name="username" />
      <button type="submit">送信</button>
    </form>
    ```

    ### 主な入力タイプ

    - **`type="text"`**: 一般的なテキスト入力
    - **`type="email"`**: メールアドレス形式の入力（バリデーションが適用されます）
    - **`type="password"`**: 入力内容を非表示にするパスワード入力
    - **`type="radio"`**: ラジオボタン、複数の選択肢から 1 つだけ選べます
    - **`type="checkbox"`**: チェックボックス、複数の選択肢から複数選べます
    - **`type="submit"`**: フォーム送信用ボタン

## HTML 属性

HTML タグには属性を追加して、要素に追加情報や特定の機能を持たせることができます。一般的な属性には以下のものがあります。

- **id**: ユニークな識別子を設定するために使用します。

  ```html
  <div id="unique-section">ユニークなセクション</div>
  ```

- **class**: 複数の要素に共通のスタイルを適用するために使用します。

  ```html
  <p class="highlighted">強調表示された段落</p>
  ```

- **style**: インラインで CSS スタイルを適用します。ただし、通常は CSS ファイルでスタイルを指定する方が推奨されます。

  ```html
  <p style="color: blue;">青色のテキスト</p>
  ```

## コンテンツモデル

HTML の**コンテンツモデル**は、各要素がどのような種類のコンテンツを含むべきかを定義するルールです。このモデルに従うことで、HTML 文書の**構造を論理的かつ意味的に整理**することができ、ブラウザや検索エンジン、支援技術がページ内容を正確に解釈しやすくなります。
コンテンツモデルを理解して使用することで、HTML 文書が**論理的でアクセスしやすい構造**を持つようになり、ブラウザや支援技術にとって解釈が容易になります。コンテンツモデルを意識しながら HTML を記述することが、ウェブ開発においての基本であり、質の高いコンテンツ作りの第一歩です。

---

### HTML の主要なコンテンツモデル

HTML では、要素を含める内容に応じていくつかの主要なコンテンツモデルに分類しています。以下がその主な内容です：

1. **フローコンテンツ（Flow Content）**
   - **概要**: HTML 文書の構造やテキスト、メディアなど、ページ全体を構成する要素が含まれます。
   - **例**: `<div>`, `<p>`, `<header>`, `<footer>`, `<section>`, `<article>`など。
   - **用途**: フローコンテンツはほとんどの HTML 要素をカバーしており、HTML ページを構成する主要な要素として広範囲に利用されます。
2. **セクショニングコンテンツ（Sectioning Content）**
   - **概要**: 文書のアウトラインを定義するために使用され、ページをセクション単位に分割します。
   - **例**: `<header>`, `<footer>`, `<nav>`, `<article>`, `<section>`, `<aside>`, `<h1>`～`<h6>`など。
   - **用途**: セクショニングコンテンツを使うことで、文書の論理構造が整理され、各セクションは独立した意味を持ち、検索エンジンや支援技術が文書の内容を理解しやすくなります。
3. **ヘディングコンテンツ（Heading Content）**
   - **概要**: 各セクションの見出しとして使用され、文書の構成を階層的に整理するために役立ちます。
   - **例**: `<h1>`, `<h2>`, `<h3>` など。
   - **用途**: セクションごとに適切な見出しを配置することで、内容の要点を伝えやすくします。また、アウトラインを形成することで、支援技術がページの内容を把握しやすくします。
4. **フレージングコンテンツ（Phrasing Content）**
   - **概要**: インライン要素を含み、文章中の一部を装飾したり、リンクや強調などの役割を果たします。
   - **例**: `<span>`, `<a>`, `<strong>`, `<em>`, `<br>`, `<img>` など。
   - **用途**: フレージングコンテンツを使うことで、テキストやインライン画像を装飾したり、リンクを追加したりすることができます。他の要素と同じ行に表示されます。
5. **埋め込みコンテンツ（Embedded Content）**
   - **概要**: 外部リソースやメディアを埋め込むために使われます。
   - **例**: `<img>`, `<audio>`, `<video>`, `<iframe>`, `<embed>`, `<object>` など。
   - **用途**: 埋め込みコンテンツを使って画像、音声、動画、外部コンテンツなどをページ内に表示し、メディアを豊かに表現できます。
6. **インタラクティブコンテンツ（Interactive Content）**
   - **概要**: ユーザーが操作可能な要素（例: リンクやフォーム）を含み、ページの機能性を高めます。
   - **例**: `<a>`, `<button>`, `<input>`, `<select>`, `<textarea>` など。
   - **用途**: インタラクティブコンテンツを利用することで、ページにリンクやフォーム、ボタンといった操作可能な要素を追加し、ユーザーがサイトと対話できるようにします。

---

### コンテンツモデルが重要な理由

コンテンツモデルは、HTML 文書を**正しく構築し、ページの意味を分かりやすく伝える**ために重要です。各要素が含むべきコンテンツのタイプや、その配置に関する制約がモデルによって定義されているため、構造が明確になります。

- **文法チェック**: コンテンツモデルに沿って正しく HTML を書くと、エラーが減り、メンテナンスがしやすくなります。
- **検索エンジン最適化（SEO）**: 適切に構造化された HTML 文書は検索エンジンに理解されやすく、検索結果に反映されやすくなります。
- **アクセシビリティ**: 支援技術がページの意味と構造を理解しやすくなり、アクセシビリティが向上します。

---

## HTML のセマンティクスタグ

HTML5 以降、**セマンティックタグ**と呼ばれるタグが追加されました。これらは、要素の意味をブラウザや検索エンジンに伝えやすくします。

- **`<header>`**: ヘッダー部分（ロゴやナビゲーションメニューなど）を表します。
- **`<nav>`**: ナビゲーションリンクをグループ化します。
- **`<section>`**: 関連する内容をまとめるセクションを定義します。
- **`<article>`**: 独立したコンテンツを表します（ニュース記事やブログ投稿など）。
- **`<footer>`**: フッター部分（連絡先情報や著作権情報など）を表します。

```html
<article>
  <header>
    <h2>記事のタイトル</h2>
  </header>
  <p>これは記事の内容です。</p>
  <footer>著作権情報など</footer>
</article>
```

## Form の基本

### input 要素

フォームの部品を表す要素。

### type 属性

フォームコントロールの型。デフォルトは submit

- text
- password
- radio
- checkbox
- file
- hidden
- search
- tel
- url
- email
- datetime
- datetime-local
- date
- month
- week
- time
- number
- range
- color
- submit：form のデータをサーバーへ送信
- button：これだけでは動作なし。イベントハンドラを渡して使う。
- reset：全てのコントロールを初期化する。

### action 属性

フォームで収集したデータを送信すべき場所（URL）を指定。

値は相対/絶対 URL 出なければいけない。

### method 属性

データを送信するために使用する HTTP メソッド（GET or POST）を指定。
